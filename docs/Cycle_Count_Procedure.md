# Cycle Count Procedure

## Overview
Cycle counting is a perpetual audit technique to verify inventory accuracy without shutting down warehouse operations. By continually counting small subsets of inventory, we maintain high accuracy and detect errors early.

## Count Frequency and ABC Analysis
We categorize inventory items based on their value and movement velocity to determine counting frequency:
- **"A" Items:** High-value or fast-moving items, counted weekly. Accuracy target: 98%.
- **"B" Items:** Moderate-value or medium-moving items, counted quarterly. Accuracy target: 95%.
- **"C" Items:** Low-value or slow-moving items, counted semi-annually. Accuracy target: 90%.

## Pre-Counting Rules
Before beginning a count in a designated zone, all physical inventory movement must be frozen. System transactions (receiving, picking, replenishment) must be closed for the items being counted to prevent systemic discrepancies during the audit.

## Variance Thresholds and Recount Rules
If a physical count differs from the Warehouse Management System (WMS) by more than 2% for A items or 5% for B/C items, a mandatory recount must be triggered by a designated Exception Resolution Specialist. 

## Adjustment Approval Process
Adjustments resulting in a financial variance of less than $500 can be approved by the Shift Manager. Adjustments exceeding $500 require approval from the Operations Director.

## Root Cause Documentation
For recurring discrepancies, a variance note with an associated reason code (e.g., mis-picks, receiving errors, unrecorded transfers) must be documented in the WMS to facilitate root cause analysis.
